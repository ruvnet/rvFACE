# ADR-0004: Exact pipeline semantics (and reconstructions where upstream is closed)

- Status: Accepted
- Date: 2026-07-02

## Context

A port is only credible if the non-neural math is bit-faithful. The upstream
pipeline mixes published Python (detector pre/post-processing) with
**unpublished** helpers (`face_util.align_vertical`, `face_pose.GetPose`,
`face_landmark.GetLandmark`). We must fix, in writing, what "exact" means
per stage.

## Decision

### Ported exactly (published upstream source is normative)

- **Detector preprocessing**: BGR→RGB, resize to 320×240 (bilinear, matching
  `cv2.resize` default), `(x − 127) / 128`, NCHW.
- **Priors**: `generate_priors` over feature maps `[[40,20,10,5],[30,15,8,4]]`,
  min_boxes `[[10,16,24],[32,48],[64,96],[128,192,256]]`, clipped to [0,1].
- **Decode**: center-form with `center_variance 0.1`, `size_variance 0.2`,
  then corner form.
- **Filtering**: softmax over 2 classes, prob threshold **0.6**, per-class
  **hard NMS** with IoU **0.3**, candidate pool **1500**, at most **750**
  picks (upstream passes `candidate_size/2` as `top_k`), then top-K faces
  (`faceMaxCount`); boxes fully inside
  the image only (`x1≥0, y1≥0, x2<W, y2<H`), matching `get_face_boundingbox`.
- **Embedder preprocessing**: aligned 128×128 BGR-order crop scaled by
  **1/256** (upstream divides by 256, not 255 — preserved deliberately),
  HWC→CHW, no mean subtraction; output L2-normalized.
- **Similarity**: `score = (Σ f1·f2 + 1) × 50`, match iff `score > 75`.

### Reconstructed (upstream source unpublished; behavior defined here)

- **Alignment** (`align_vertical(image, W, H, out, 128, 128, 3, landmarks,
  48, 64, 40)`): reconstructed as the standard eyes-level similarity
  transform implied by the call signature: rotate+scale+translate the source
  so that the eye centers (means of landmarks 36–41 and 42–47) become
  horizontal with midpoint at **(64, 48)** and inter-eye distance **40 px**
  in a **128×128** output, bilinear sampling, edge clamp.
- **Landmark extraction** (`GetLandmark`): follows the checkpoint's
  reference preprocessing (cunjian's): square-expand the detector box,
  crop, resize to the model's input size, scale to [0,1] (grayscale for the
  upstream 64×64 variant), output 136 values scaled back into the crop's
  coordinate frame.
- **Head pose** (`GetPose`): yaw/pitch/roll estimated from the 68 landmarks
  against a canonical 3D face template via orthographic least-squares (EPnP
  is overkill without camera intrinsics upstream never had). Roll is
  additionally exposed directly from the eye-line angle.

Every reconstruction is labeled `// reconstruction: upstream face_util is
unpublished` at the definition site.

## Consequences

- All "ported exactly" items get golden-fixture tests generated by running
  the actual upstream Python (ADR-0006); reconstructions get property tests
  (e.g. post-alignment eye positions land at (44,48) and (84,48) ±ε) and
  end-to-end score checks instead.
- The 1/256 quirk and threshold 75 are kept even where "better" choices
  exist; rvFACE is a port, not a redesign. Any deviation requires a new ADR.
